# Create Implementation Plan

Create a new implementation plan for a feature or task.

## Required Skills

Before starting, invoke these skills in order:
1. `brainstorming` - Refine the idea through questioning
2. `writing-plans` - Create detailed implementation plan

## Process

1. **Understand the request** - Ask clarifying questions
2. **Explore approaches** - Present 2-3 options with trade-offs
3. **Design incrementally** - Present design in 200-300 word sections
4. **Document** - Save plan to `docs/plans/YYYY-MM-DD-<feature>.md`
5. **Update index** - Add entry to `docs/plans/main.md` under "Current Plans" with status "Pending"

## Updating main.md

After creating a plan, add a row to the Current Plans table in `docs/plans/main.md`:

```markdown
| [YYYY-MM-DD-feature-name](./YYYY-MM-DD-feature-name.md) | Pending | Short description |
```

## Output Location

Plans are saved to: `docs/plans/`

## Arguments

$ARGUMENTS - Description of what to build (optional, will ask if not provided)
