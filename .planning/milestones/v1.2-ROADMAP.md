# Milestone v1.2: Cleanup & Quality

**Status:** SHIPPED 2026-03-04
**Phases:** 14-16
**Total Plans:** 7

## Overview

Tighten what's already built — better abort behavior, cleaner codebase, better user feedback.

## Phases

### Phase 14: Abort Propagation

**Goal**: Clicking abort actually stops all in-flight OpenRouter LLM calls within seconds and the workflow shows "canceled" not "failed"
**Depends on**: Nothing (first phase in v1.2)
**Requirements**: ABORT-01, ABORT-02, ABORT-03, ABORT-04
**Plans**: 2 plans

- [x] 14-01-PLAN.md — Backend abort infrastructure: cancel endpoint, signal threading, iteration boundary checks
- [x] 14-02-PLAN.md — Frontend abort UX: confirmation dialog, aborting state, visual feedback

**Success Criteria:**
1. User clicks abort during a solve and all pending LLM API calls stop within seconds
2. After aborting, the workflow status shows amber "aborted" state, not red error/failed
3. User can abort during verify/improve loop and next iteration does not start
4. Abort works reliably even if browser-to-server signal is unreliable (cancel endpoint fallback)

### Phase 15: File Refactoring

**Goal**: Oversized source files split into focused, single-responsibility modules with zero behavior changes
**Depends on**: Phase 14
**Requirements**: REFAC-01, REFAC-02, REFAC-03, REFAC-04
**Plans**: 3 plans

- [x] 15-01-PLAN.md — Extract workflow step definitions to individual files under steps/
- [x] 15-02-PLAN.md — Split trace-event-card.tsx sub-components into trace/ directory
- [x] 15-03-PLAN.md — Extract page.tsx hooks to dedicated hook files

**Success Criteria:**
1. `workflow.ts` is a short composition file (~30 lines) with steps in `steps/`
2. `trace-event-card.tsx` sub-components in focused files under `components/trace/`
3. `page.tsx` smaller with extracted hooks; solver page works identically
4. `npx tsc --noEmit` passes with no new errors

### Phase 16: Toast Notifications

**Goal**: Users get non-blocking feedback for workflow lifecycle events without watching the UI constantly
**Depends on**: Phase 15
**Requirements**: TOAST-01, TOAST-02, TOAST-03, TOAST-04, TOAST-05, TOAST-06, TOAST-07
**Plans**: 2 plans

- [x] 16-01-PLAN.md — Install Sonner, create blueprint-themed toast component, wire lifecycle toasts
- [x] 16-02-PLAN.md — Add backend cost tracking with $1 boundary events, wire cost warning toasts

**Success Criteria:**
1. Toast when solve starts, and distinct toasts for complete/abort/error
2. Cost-warning toast when cumulative API spend crosses configurable thresholds
3. Toasts styled consistently with blueprint/cyanotype design system
4. Stable IDs prevent duplicates in React Strict Mode

## Milestone Summary

**Key Decisions:**
- Build order: abort → refactor → toasts (avoids merge conflicts on shared files)
- `any` type for Run in activeRuns Map (only used for cancel() calls)
- Replaced browser confirm() with shadcn AlertDialog for abort confirmation
- Step files use `../` for sibling workflow imports and `@/` for external imports
- Moved ToolCallRenderer to specialized-tools.tsx to avoid circular dependency
- Cost accumulates per-step (not per-workflow) to avoid modifying workflow-schemas.ts

**Issues Resolved:**
- Abort signal not propagating to in-flight LLM calls
- Large monolith files (workflow.ts 1500 lines, trace-event-card.tsx 898 lines)
- Missing workflow lifecycle feedback (no toasts)
- Abort dialog causing layout shift (quick task fix)

**Issues Deferred:**
- Custom toast variants with progress indicators (OBS-01)
- Toast notification preferences (OBS-02)
- Additional file size audit (QUAL-01)

**Technical Debt Incurred:**
- `any` type for Run in activeRuns Map (minor; only used for cancel())
