Replace the Phase 20 stub in synthesizer.md with a complete self-contained system prompt. Follow the same structure established by extractor.md and hypothesizer.md (Domain Context, Input, Task, Output Format, Do NOT, Error Handling).

The synthesizer merges vocabulary and rules from multiple verified perspectives into a single consolidated solution. Key design points from PIPELINE.md and CONTEXT.md:

**Domain Context:** Brief role description -- merges best-performing rules from multiple perspectives into a coherent solution.

**Input** (5 items the orchestrator provides):
1. Paths to all perspective files (perspective-N.md) from the current round
2. Paths to corresponding verification files (verification-N.md) -- these contain pass rates and diagnostic notes
3. Path to problem.md (the original extracted problem for reference)
4. Output file path for solution.md
5. For Round 2+, path to previous solution.md as baseline

**Task -- Merging Process** (from PIPELINE.md Section 4d):
1. Read all perspective files and their verification results
2. Rank perspectives by pass rate (from their verification ## Summary)
3. Start with the highest-scoring perspective's vocabulary and rules as the base
4. Add complementary rules from lower-scoring perspectives (rules covering phenomena the base does not address)
5. Resolve conflicts via score-weighted resolution: same phenomenon -> favor higher-scoring perspective; complementary -> include both; conflicting -> favor higher pass rate
6. Refine vocabulary: if rule merging reveals issues, update vocabulary entries accordingly
7. Test merged rules mentally against dataset sentences to check for contradictions

**Output Format:** Write solution.md matching the template in workspace-format.md:
- `# Solution` header
- `## Vocabulary` table with Foreign Form, Meaning, Type, Notes, Source columns (Source cites which perspective(s) contributed each entry)
- `## Rules` sections with title, description, Evidence, Source (with pass rates), Confidence

**Do NOT (6-8 items):**
- Do NOT invent rules not present in any perspective -- only merge, select, and refine existing rules
- Do NOT discard all rules from a perspective just because it has a lower pass rate -- it may have complementary rules the higher-scoring perspective lacks
- Do NOT include contradictory rules -- resolve conflicts before writing
- Do NOT duplicate vocabulary entries with the same foreign form -- pick the best gloss and cite all sources
- Do NOT answer the questions -- only produce the merged ruleset and vocabulary
- Do NOT reference other files besides the ones provided as input paths

**Error Handling:** Same pattern as extractor/hypothesizer -- write partial solution.md with ## Error section + append to errors.md.

**Preserve frontmatter** exactly as-is (name, description, tools, model: opus). Only replace the body below the frontmatter closing `---`.

Target: 80-120 lines of prompt body (comparable to extractor at ~112 lines).

**Preserve frontmatter** exactly: name: solve, description, disable-model-invocation: true, argument-hint: "[file-path]".

**The orchestrator has these major sections:**

### Step 1: Get Problem Input (ORCH-01, ORCH-02)

- If `$ARGUMENTS` contains a file path, read that file to get the problem text
- If `$ARGUMENTS` is empty, ask the user: "Please paste your Linguistics Olympiad problem text, or provide a file path."
- If the user response looks like a file path (starts with `/` or `./`, or ends with `.md`/`.txt`), read that file
- Otherwise treat the response as inline problem text

### Step 2: Create Workspace (ORCH-04)

- Generate timestamp via `date +"%Y-%m-%d_%H-%M-%S"`
- Create workspace: `claude-code/workspace/{timestamp}/` and `claude-code/workspace/{timestamp}/hypotheses/round-1/`
- Write raw problem text to `{workspace}/problem-raw.md`
- Store the workspace path as `WORKSPACE` for all subsequent steps

### Step 3: Extract Problem Structure (ORCH-03, ORCH-04)

- Print: `"Extracting problem structure..."`
- Use the **extractor** agent:
  - Read problem from: `{WORKSPACE}/problem-raw.md`
  - Write extracted problem to: `{WORKSPACE}/problem.md`
- After completion, check that `{WORKSPACE}/problem.md` exists
- If it does not exist: print error and abort

### Step 4: Multi-Perspective Hypothesis Loop (ORCH-03, ORCH-04, ORCH-05)

This is the core loop, running up to 3 rounds.

**For each round N (1 to 3):**

#### Step 4a: Generate Perspectives (inline, no subagent)

**Round 1:** The orchestrator itself reads problem.md and generates 3 diverse linguistic perspectives. Print: `"Round {N}: Generating 3 perspectives..."`. For each perspective, create a name and a 2-3 sentence focus description covering what linguistic angle to explore. Choose from: morphological analysis, syntactic analysis, phonological analysis, agreement/concord patterns, semantic/pragmatic features, orthographic patterns. Ensure all 3 are genuinely different angles.

**Round 2+:** The orchestrator reads the verification.md from the previous round, identifies which rules failed and which sentences had errors. Print: `"Round {N}: Analyzing gaps and generating targeted perspectives..."`. Generate 2-3 targeted perspectives that address the specific failures. Create the directory `{WORKSPACE}/hypotheses/round-{N}/`.

#### Step 4b: Dispatch Hypothesizers (sequential, ORCH-03)

For each perspective P (1 to count):

- Print: `"Generating perspective {P} of {count}: {perspective_name}..."`
- Use the **hypothesizer** agent:
  - Read problem from: `{WORKSPACE}/problem.md`
  - Assigned perspective: `{name} -- {description}`
  - Round: `{N}`, Perspective: `{P}`
  - Write output to: `{WORKSPACE}/hypotheses/round-{N}/perspective-{P}.md`
  - For Round 2+: Also provide `"Read baseline rules and vocabulary from: {WORKSPACE}/solution.md"`
- After completion, check that the output file exists
- If it does not exist on first attempt: retry once
- If retry also fails: print `"Perspective {P} failed -- continuing with remaining perspectives"`, append error to `{WORKSPACE}/errors.md`, continue to next perspective

#### Step 4c: Dispatch Verifiers (sequential, ORCH-03)

For each perspective P that produced an output file:

- Print: `"Verifying perspective {P}: {perspective_name}..."`
- Use the **verifier** agent:
  - Read hypothesis from: `{WORKSPACE}/hypotheses/round-{N}/perspective-{P}.md`
  - Read problem from: `{WORKSPACE}/problem.md`
  - Write verification results to: `{WORKSPACE}/hypotheses/round-{N}/verification-{P}.md`
- After completion, check that the verification file exists
- If it does not exist: print `"Verification of perspective {P} failed -- continuing"`, append error to `{WORKSPACE}/errors.md`

#### Step 4d: Read Pass Rates and Print Summary

For each perspective that has a verification file:
- Read the `## Summary` section of `verification-{P}.md`
- Extract the `Pass rate: {N}%` value
- Print: `"Round {N} results: {name1} {rate1}%, {name2} {rate2}%, {name3} {rate3}%"`

If no perspectives produced valid verification results, abort with error.

#### Step 4e: Dispatch Synthesizer (ORCH-05)

- Print: `"Synthesizing best rules from all perspectives..."`
- Use the **synthesizer** agent:
  - Read all perspective files: `{WORKSPACE}/hypotheses/round-{N}/perspective-*.md`
  - Read all verification files: `{WORKSPACE}/hypotheses/round-{N}/verification-*.md`
  - Read problem from: `{WORKSPACE}/problem.md`
  - For Round 2+: Read previous solution from: `{WORKSPACE}/solution.md`
  - Write merged solution to: `{WORKSPACE}/solution.md`
- After completion, check that `{WORKSPACE}/solution.md` exists
- If it does not exist: use the highest-pass-rate perspective file as a fallback solution

#### Step 4f: Convergence Check

- Print: `"Checking convergence..."`
- Use the **verifier** agent:
  - Read hypothesis from: `{WORKSPACE}/solution.md`
  - Read problem from: `{WORKSPACE}/problem.md`
  - Write verification results to: `{WORKSPACE}/verification.md`
- After completion, read `{WORKSPACE}/verification.md` and check the `## Summary` section
- If pass rate is 100% (all rules pass, all sentences pass):
  - Print: `"Converged! All rules pass verification."`
  - Break out of the loop
- If not converged and round < 3:
  - Print: `"Round {N} pass rate: {rate}%. Starting round {N+1}..."`
  - Continue to next round
- If not converged and round = 3:
  - Print: `"Maximum rounds reached. Using best result (pass rate: {rate}%)."`
  - Break out of the loop
