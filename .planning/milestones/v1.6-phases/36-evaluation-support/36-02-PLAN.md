Modify `src/app/evals/page.tsx` to add a provider filter dropdown in the run history table:

1. **Add filter state**: In the `EvalsPage` component, add state for the provider filter:
```typescript
const [providerFilter, setProviderFilter] = useState<string>('all');
```

2. **Derive unique provider modes from data**: Compute the list of unique provider modes from the loaded runs:
```typescript
const providerModes = [...new Set(runs.map((r) => r.providerMode))].sort();
```

3. **Compute filtered runs**: Filter runs based on the selected provider:
```typescript
const filteredRuns = providerFilter === 'all'
  ? runs
  : runs.filter((r) => r.providerMode === providerFilter);
```

4. **Add filter dropdown in the run history header**: Inside the `BlueprintCard` that contains "Run History", modify the `h2` area to include a filter dropdown. Replace the existing `<h2>` with a flex container:
```tsx
<div className="mb-3 flex items-center gap-3">
  <h2 className="font-heading text-base text-foreground">Run History</h2>
  {providerModes.length > 1 && (
    <select
      value={providerFilter}
      onChange={(e) => setProviderFilter(e.target.value)}
      className="hover-hatch-cyan border border-border-subtle bg-transparent px-2 py-1 text-xs uppercase tracking-wider text-muted-foreground focus:border-accent focus:text-foreground focus:outline-none"
    >
      <option value="all">All providers</option>
      {providerModes.map((pm) => (
        <option key={pm} value={pm}>
          {pm}
        </option>
      ))}
    </select>
  )}
</div>
```

The dropdown only renders when there are 2+ distinct provider modes in the data (no point showing a filter with one option). Style follows DESIGN.md: no rounded corners (inherited from global radius: 0), uppercase tracking-wider label text, border-subtle borders, transparent bg, cyan focus ring.

5. **Use filteredRuns in the table**: Replace `runs.map((run) => ...)` in the `TableBody` with `filteredRuns.map((run) => ...)`. This is the only table mapping that needs to change -- the `runs` array is still used for the `providerModes` derivation.

const selectedRun = filteredRuns.find((r) => r.id === selectedRunId) ?? null;

This way if a selected run is filtered out, the detail panel simply hides -- no confusing state.
