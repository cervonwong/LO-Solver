Add a new `streamWithRetry` function alongside the existing `generateWithRetry`. Do NOT modify or remove `generateWithRetry` — it's still used by tool files (rule-tester-tool.ts, sentence-tester-tool.ts) that don't need streaming.

**Function signature:**
```typescript
interface StreamWithRetryOptions<TOptions> {
  prompt: string;
  options?: TOptions;
  timeoutMs?: number;       // Default: 600_000 (10 minutes)
  maxRetries?: number;      // Default: 2 (3 total attempts)
  abortSignal?: AbortSignal;
  onTextChunk?: (chunk: string) => void;  // Callback for real-time text streaming
}

export async function streamWithRetry<TOptions extends Parameters<Agent['stream']>[1]>(
  agent: Agent,
  opts: StreamWithRetryOptions<TOptions>,
): Promise<FullOutput<any>>
```

**Implementation details:**

1. Same retry/timeout structure as `generateWithRetry` (loop with attempt counter, AbortController merging, hard timeout via Promise.race, linear backoff on retry).

2. Inside the try block:
   - Call `agent.stream(prompt, { ...options, abortSignal: mergedSignal })` to get `MastraModelOutput`.
   - Consume the `textStream` ReadableStream, calling `onTextChunk(chunk)` for each chunk. This enables real-time text streaming to the frontend.
   - After the stream is consumed, call `.getFullOutput()` to get the complete result.
   - Return the `FullOutput` object (which has `.text`, `.object`, `.reasoning`, `.toolCalls`, etc.).

3. Empty response check: same logic as generateWithRetry — if `structuredOutput` is in options, check `result.object`; otherwise check `result.text`.

4. Import `FullOutput` type from `@mastra/core` (check the actual export path: `import type { FullOutput } from '@mastra/core/stream'` or wherever it's exported from — look in node_modules).

5. The `onTextChunk` callback is optional. When not provided, still consume the textStream (to drive the stream forward) but don't do anything with chunks.

**Why this approach:** The user decided to stream agent reasoning text in real-time. By consuming textStream and forwarding chunks via callback, the workflow step can emit `AgentTextChunkEvent` events to the frontend while the agent is still generating. The `getFullOutput()` call at the end provides the complete result for schema validation and logging, matching how `generateWithRetry` is currently used.
