**Internal state:**
- `inputValue: string` — the text in the input field
- `isEditing: boolean` — whether the input is in edit mode
- `showClearConfirm: boolean` — inline confirmation state for clearing

**Behavior on open (`useEffect` on `open` prop):**
- If a key exists (from `useApiKey`): set `isEditing = false`, `inputValue = ''`, `showClearConfirm = false`
- If no key: set `isEditing = true`, `inputValue = ''`, `showClearConfirm = false`

**Read-only state (key exists, not editing):**
- Show masked preview as a read-only display: `sk-...{last4chars}` (or full key if shorter than 8 chars)
- Show an "Edit" button that sets `isEditing = true` and clears `inputValue`

**Edit state (editing or no key):**
- Show the shadcn `<Input>` component with `type="text"` (per user decision: no password masking), placeholder "sk-or-..." or similar
- Input value bound to `inputValue`

**Dialog structure (follow abort dialog styling pattern):**

- `DialogHeader`:

  - `DialogDescription`: brief description about entering an OpenRouter API key
- Body area: the input or masked preview (described above)
