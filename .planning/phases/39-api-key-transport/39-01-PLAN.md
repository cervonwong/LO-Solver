<objective>
Remove the API key from all persisted workflow state and switch the backend to header-based key transport.

Purpose: The API key currently leaks to LibSQL via workflow state snapshots (both `workflowStateSchema.apiKey` and `rawProblemInputSchema.apiKey` are persisted in snapshot `context.input`). This plan removes the key from both schemas and propagates it through the non-persisted workflow-level `RequestContext` parameter on `run.stream()`. It also updates both API route handlers to read the key from the `x-openrouter-key` HTTP header.

Output: All backend files updated -- schemas clean, routes reading from headers, steps reading from workflow RequestContext. TypeScript compiles cleanly.
</objective>
