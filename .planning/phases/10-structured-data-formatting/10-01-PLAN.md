Output: A reusable `LabeledList` component and updated `AgentToolCallCard` / `ToolCallDetail` components that use it instead of `jsonMarkdown()`.
</objective>

  <files>src/components/labeled-list.tsx</files>
  <action>
Create a new React component `LabeledList` that renders a JSON object as stacked key-value rows.

Props interface:
```typescript
interface LabeledListProps {
  data: Record<string, unknown>;
  label?: string;           // Optional section header (e.g., "Input", "Result")
  className?: string;       // Additional classes for the container
}
```

Rendering rules (from CONTEXT.md decisions):
1. **Stacked rows**: Each key-value pair on its own line
2. **Label column**: Key names rendered in `text-muted-foreground text-[11px]` (matching trace panel font size), left-aligned
3. **Value column**: Values in regular `text-[11px]` foreground text, right of label
4. **Layout**: Use a CSS grid with `grid-template-columns: auto 1fr` so the label column auto-sizes to the longest label in that section. Gap between label and value: `gap-x-3 gap-y-0.5`.
5. **Section header**: If `label` prop is provided, render it as `<span className="text-[10px] font-medium text-muted-foreground uppercase">{label}</span>` above the grid (same style as "Reasoning" header in AgentCard).
6. **Nested objects** (depth <= 2): Indent sub-keys under the parent key. Use recursive rendering with a depth counter. Parent key renders as a label with no value; children indent with `pl-4`.
7. **Beyond depth 2**: Render the value as a compact inline JSON string (`JSON.stringify(value)`) in `text-muted-foreground font-mono text-[10px]`.
8. **Arrays**: Render as numbered items `[0]`, `[1]`, etc., one per line. Object items in arrays get the indented sub-key treatment (if within depth limit).
9. **Long string values**: Wrap naturally, no truncation. Use `break-words` or `break-all` as needed.
10. **Edge cases**: `null` renders as italic "null", `undefined` as italic "undefined", empty objects as italic "empty", booleans and numbers render as-is.

Do NOT add Streamdown or code block rendering — this is a plain React component with Tailwind classes.
