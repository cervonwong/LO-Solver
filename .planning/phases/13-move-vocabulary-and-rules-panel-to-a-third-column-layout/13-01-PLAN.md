**2. Update `src/app/globals.css`:**

Add a utility class for panel flex-grow transitions. The `react-resizable-panels` library sets `flex-grow` as an inline style on panel elements. Adding a CSS transition on `flex-grow` will animate size changes when the imperative API is used.

```css
/* Animated panel resize transition — applied during workflow-start layout shift */
.panel-transition [data-panel] {
  transition: flex-grow 600ms cubic-bezier(0.4, 0, 0.2, 1);
}
```

**3. Restructure `src/app/page.tsx` layout:**

The core change: move from a 2-column layout (left | right-with-vertical-stack) to a conditional 2-or-3-column layout (left | trace | vocab+rules).

**Import additions:**
- Import `useGroupRef` from `react-resizable-panels` (the hook, not from the shadcn wrapper)

**New state/refs (add near existing state declarations):**
- `const groupRef = useGroupRef()` — imperative handle for the outer horizontal ResizablePanelGroup
- `const isLargeScreen = useMediaQuery('(min-width: 1024px)')` — true above lg breakpoint
- `const hasAnimated = useRef(false)` — tracks if the workflow-start animation has already fired
- `const [isTransitioning, setIsTransitioning] = useState(false)` — controls CSS transition class

**Determine if 3-column mode is active:**
```typescript
const showThirdColumn = hasStarted && isLargeScreen;
```

**Animated transition logic (useEffect):**

When `hasStarted` becomes true AND `isLargeScreen` is true AND `hasAnimated.current` is false:
1. Set `isTransitioning` to `true` (adds `.panel-transition` class to the group wrapper)
2. Use `requestAnimationFrame` to call `groupRef.current?.setLayout()` with the 3-column sizes: `{ 'input-panel': 20, 'trace-panel': 50, 'vocab-rules-panel': 30 }`
3. After a 650ms timeout (slightly longer than the 600ms transition), set `isTransitioning` to `false` to remove the transition class (prevents interfering with manual drag resizing)
4. Set `hasAnimated.current = true`

When the screen drops below 1024px (`!isLargeScreen`), reset `hasAnimated.current = false` so the animation can replay if the user resizes back above 1024px and the workflow is still running.

**Important implementation details:**

- The panel IDs (`input-panel`, `trace-panel`, `vocab-rules-panel`) are required for the `groupRef.setLayout()` imperative API to work. They must match the keys in the layout object.
- `defaultSize` values on the left panel change based on `showThirdColumn`: `"50%"` when 2-column, `"20%"` when 3-column. This ensures correct initial proportions in each mode.
- The `handleReset` callback should also reset `hasAnimated.current = false` so the animation replays on next solve.
- When `showThirdColumn` is false and `hasStarted` is true (narrow screen), the old stacked layout renders inside the trace panel. This is the responsive collapse behavior.
- The responsive collapse is instant (no animation) because `isTransitioning` is only set during the one-time workflow-start transition.
- The wrapping div with `panel-transition` class must contain the ResizablePanelGroup so the CSS descendant selector `.panel-transition [data-panel]` targets the panel elements.

**Caution with conditional panels:**
The `react-resizable-panels` library may re-mount when the number of panels changes (going from 2 to 3 panels in the group). This is expected behavior. The `defaultSize` props handle the initial sizing for each configuration. The animation via `setLayout()` fires after the 3-panel configuration mounts, smoothly resizing from the `defaultSize` values to the target proportions.

If the library throws errors when conditionally rendering panels, an alternative approach: always render 3 panels but set the third panel to `collapsible={true} collapsedSize={0}` and use `panelRef.expand()` / `panelRef.collapse()` to show/hide. Try the conditional approach first as it is simpler.
