**1. Create `/api/claude-auth` endpoint** (`src/app/api/claude-auth/route.ts`):
- GET handler that runs `claude auth status --json` via `child_process.execFile` (promisified).
- Use 5-second timeout. Parse JSON output.
- Return `{ authenticated: boolean, email?: string, subscriptionType?: string }`.
- On ENOENT or parse error, return `{ authenticated: false, error: string }`.
- Always return 200 (never throw to client -- the auth status itself conveys the state).

**2. Create `useClaudeAuth` hook** (`src/hooks/use-claude-auth.ts`):
- `export function useClaudeAuth(enabled: boolean)` -- returns `{ authenticated: boolean, loading: boolean, email?: string }`.
- When `enabled` is true: fetch `/api/claude-auth` immediately, then poll every 20 seconds.
- When `enabled` is false: return `{ authenticated: false, loading: false }` and do not poll.
- Clean up interval on unmount or when `enabled` changes.

**3. Expand client ProviderMode** (`src/hooks/use-provider-mode.ts`):
- Update `ProviderMode` type to 4 values: `'openrouter-testing' | 'openrouter-production' | 'claude-code-testing' | 'claude-code-production'`.
- Update `VALID_MODES` array to match.
- Add migration in `getSnapshot()`: if stored value is `'claude-code'`, migrate to `'claude-code-testing'` and update localStorage.
- Export `isClaudeCodeMode` and `isOpenRouterMode` helper functions (client-side duplicates -- do NOT import from server module `openrouter.ts`):
  ```typescript
  export function isClaudeCodeMode(mode: ProviderMode): boolean {
    return mode === 'claude-code-testing' || mode === 'claude-code-production';
  }
  export function isOpenRouterMode(mode: ProviderMode): boolean {
    return mode === 'openrouter-testing' || mode === 'openrouter-production';
  }
  ```

**4. Expand ProviderModeToggle** (`src/components/provider-mode-toggle.tsx`):
- Update local `ProviderMode` type to 4 values.
- Update `OPTIONS` array to 4 items:
  ```typescript
  { value: 'openrouter-testing', label: 'Test' },
  { value: 'openrouter-production', label: 'Prod' },
  { value: 'claude-code-testing', label: 'CC Test' },
  { value: 'claude-code-production', label: 'CC Prod' },
  ```
- Add toast notification on mode switch using Sonner: when mode changes, show `toast('Switched to {label}')`. For CC modes, append ". Checking authentication..." to the toast message.
- Import `toast` from `sonner`. Import `isClaudeCodeMode` from `@/hooks/use-provider-mode`.

**5. Update CreditsBadge** (`src/components/credits-badge.tsx`):
- Add `providerMode` to the props interface (required prop).
- Import `isClaudeCodeMode` from `@/hooks/use-provider-mode`.
- Import `useClaudeAuth` from `@/hooks/use-claude-auth`.
- When `isClaudeCodeMode(providerMode)` is true:
  - Call `useClaudeAuth(true)` for auth polling.
  - Render as a non-interactive `<div>` (not `<button>`), remove `onClick` and `cursor-pointer`.
  - Top row: green checkmark + "Claude Code" text when authenticated; amber warning + 'Run `claude login`' text with pulse animation when not.
  - Bottom row: show "Subscription" as static text. This satisfies PROV-06 as a baseline. Real cost data flows through existing cost update events from Plan 01 and appears in the trace panel.
  - Do NOT call the existing `/api/credits` polling when in CC mode.
- When `isOpenRouterMode(providerMode)` is true (or when providerMode is undefined for backward compat):
  - Keep all existing behavior unchanged (polling, key icon, dollar balance, onClick).
- Maintain the `onServerKeyStatus` callback for OpenRouter modes.

**6. Update layout-shell.tsx**:
- Import `useProviderMode` in NavBar.
- Get `[providerMode]` from `useProviderMode()`.
- Pass `providerMode` to `CreditsBadge` as a prop.
- When `isClaudeCodeMode(providerMode)`: do NOT pass `onClick` to CreditsBadge (or pass undefined). The badge is non-interactive for CC modes.
- When `isOpenRouterMode(providerMode)`: pass `onClick={() => setApiKeyDialogOpen(true)}` as before.
- Import `isClaudeCodeMode, isOpenRouterMode` from `@/hooks/use-provider-mode`.

**7. Update solve flow guard** (`src/app/page.tsx`):
- Import `useProviderMode`, `isClaudeCodeMode` from `@/hooks/use-provider-mode`.

  - If OpenRouter mode: existing `requiresKeyEntry` logic unchanged.
