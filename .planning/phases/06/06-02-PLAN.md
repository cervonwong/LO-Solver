Create `src/components/rules-panel.tsx` following the vocabulary-panel.tsx pattern closely (same frosted header, ScrollArea body, empty state).

**Props interface:**
```typescript
interface RuleEntry {
  title: string;
  description: string;
  confidence?: string;
  testStatus?: 'pass' | 'fail' | 'untested';
}

interface ActivityEvent {
  label: string;       // e.g., "+2 added", "1 updated", "Rules cleared"
  variant: 'add' | 'update' | 'remove' | 'clear';
  timestamp: number;   // Date.now() when event occurred
}

interface RulesPanelProps {
  rules: RuleEntry[];
  activityEvents: ActivityEvent[];
  isRunning: boolean;
}
```

**Table layout** (per user decision):
- Columns: `#` (ordinal, 1-indexed) | `Title` | `Description` (truncated to ~60 chars with ellipsis) | `Confidence` (badge) | `Status` (test indicator)
- Use shadcn Table components (same as vocabulary-panel.tsx)

**Expandable rows:**
- Wrap each TableRow in a Collapsible. Clicking the row toggles a CollapsibleContent below it that shows the full description text.
- Use a small chevron icon or visual indicator that the row is expandable.

**Test status indicators** (per user decision: green checkmark for pass, red X for fail, neutral dash for untested):
- `pass`: Small green checkmark SVG icon
- `fail`: Small red X SVG icon
- `untested`: Gray dash or empty

**Confidence badges** (same pattern as results-panel.tsx):
- HIGH: `variant="outline"` (accent-colored)
- MEDIUM: `variant="secondary"`
- LOW: `variant="destructive"`

**Animations** (per user decision):
- New rules: Use `animate-slide-in-row` class (already exists in globals.css)
- Updated rules: Use `animate-highlight-flash` class — add a new CSS class that applies the existing `highlight-flash` keyframes. Use a distinct color from the add animation (e.g., amber/yellow tint: `rgba(245, 158, 11, 0.15)`).
- Removed rules: Add new keyframes `strikethrough-fade` — first apply `text-decoration: line-through` then fade opacity to 0 over 400ms. Add class `animate-strikethrough-fade`.
- Bulk clear: All rules fade out together. The "Rules cleared" activity chip handles the messaging.

To trigger animations on state changes, the parent (page.tsx, wired in Plan 06-04) will need to track which rules were just added/updated/removed and pass appropriate CSS classes. For now, apply `animate-slide-in-row` to all rows by default (same as vocabulary-panel). Plan 06-04 will handle the differential animation logic when wiring state.

**Header:**
- Icon: Use a rules/list SVG icon (similar weight to vocabulary panel's book icon)

- Count: "{rules.length} {rules.length === 1 ? 'rule' : 'rules'}"
- Right side: `<RollingActivityChips>` component (from Task 2)

**Create `src/components/rolling-activity-chips.tsx`:**

Per user decision: rolling activity chips replace cumulative mutation badges. Max 3 chips visible at once. Oldest chip fades out as newest animates in (e.g., "+2 added", "1 updated", "Rules cleared").

```typescript
interface ActivityEvent {
  label: string;        // e.g., "+2 added", "1 updated"
  variant: 'add' | 'update' | 'remove' | 'clear';
  timestamp: number;    // Date.now()
}

interface RollingActivityChipsProps {
  events: ActivityEvent[];
  maxVisible?: number;  // Default: 3
}
```

**Implementation:**
- Accept an array of `ActivityEvent` objects. Display only the most recent `maxVisible` (default 3).
- Each chip is a small Badge with color based on variant:
  - `add`: border-foreground text-foreground (same as current vocab added badge)
  - `update`: border-status-warning text-status-warning
  - `remove` / `clear`: border-destructive text-destructive
- Chips animate in with a slide-in from the right and animate out (oldest) with a fade-left.
- Add CSS keyframes to globals.css:
  ```css
  @keyframes chip-in {
    from { opacity: 0; transform: translateX(8px); }
    to { opacity: 1; transform: translateX(0); }
  }
  @keyframes chip-out {
    from { opacity: 1; transform: translateX(0); }
    to { opacity: 0; transform: translateX(-8px); }
  }
  .animate-chip-in {
    animation: chip-in 200ms ease-out both;
  }
  .animate-chip-out {
    animation: chip-out 200ms ease-out forwards;
  }
  ```
- Auto-expire chips after ~8 seconds (configurable). Use a useEffect timer that removes stale events. This keeps the chip area clean when the solver is idle between operations. Claude's discretion on exact timing.
