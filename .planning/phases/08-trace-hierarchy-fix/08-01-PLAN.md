<task type="auto">
  <name>Task 3: Diagnose and fix parentId emission for tool-call events</name>
  <files>src/mastra/workflow/workflow.ts, src/mastra/workflow/request-context-helpers.ts</files>
  <action>
Investigate and fix the root cause of tool-call events arriving without a correct parentId.

**Investigation:** The `emitToolTraceEvent` helper (request-context-helpers.ts:158) reads `parent-agent-id` from the requestContext passed to the tool. The workflow sets `parent-agent-id` on the correct requestContext (`mainRequestContext`, `perspectiveRequestContext`, `verifyRequestContext`, `convergenceRequestContext`) before each agent call.

The root cause is that tools receive their requestContext from the agent's `generate()` call via `options.requestContext`. When tools call `emitToolTraceEvent(ctx.requestContext, ...)`, they get the same requestContext instance that was passed to the agent. Since the workflow correctly sets `parent-agent-id` on these requestContext instances before the agent call, the parentId should flow through.

**Diagnostic approach:** Add temporary `console.log` statements in `emitToolTraceEvent` to log:
- The `parentAgentId` value read from requestContext
- The tool event type and toolName
- Whether parentId is being injected

Run the app with `npm run dev:next` and `npm run dev:mastra` separately, trigger a solve, and check the Mastra dev server console output.

**Fix the root cause** based on diagnosis. Possible fixes:
- If `parent-agent-id` is never set because the requestContext instance differs: ensure the same requestContext instance passed to `options.requestContext` in `streamWithRetry` is the one where `parent-agent-id` is set.
- If `parent-agent-id` is cleared before tools finish executing: move the `set('parent-agent-id', undefined)` call to AFTER the agent-end event emission (it already appears to be after, but verify timing with async execution).
- If some agents don't have tools (extract, answer steps): these agents don't call tools, so no fix needed there. Focus on the multi-perspective-hypothesis step agents that DO call tools (hypothesizer, verifier-orchestrator, hypothesis-synthesizer).

**After fixing:** Remove the diagnostic `console.log` statements. Do NOT remove the `activeAgentStack` fallback yet — that is handled in Plan 02.

In `emitToolTraceEvent`, if `parentAgentId` is falsy (empty string or undefined), keep the current behavior of setting `parentId: ''` — the frontend will handle orphans in Plan 02.
