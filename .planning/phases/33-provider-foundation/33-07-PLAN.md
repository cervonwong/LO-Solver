  <action>
Modify `streamWithRetry` to detect when (a) the provider mode is `claude-code` AND (b) structuredOutput is present in options. When both conditions are true, delegate to `generateWithRetry` instead of streaming. This avoids the provider bug where `result.object` is null after stream consumption.

**Detection logic** (add at the top of `streamWithRetry`, before the retry loop):

```typescript
// Claude Code provider does not populate result.object when streaming with structuredOutput.
// Fall back to generateWithRetry which uses agent.generate() (non-streaming) for correct structured output.
const hasStructuredOutput = options && typeof options === 'object' && 'structuredOutput' in options;
const requestContext = options && typeof options === 'object' && 'requestContext' in options
  ? (options as Record<string, unknown>).requestContext
  : undefined;
const providerMode = requestContext && typeof requestContext === 'object' && 'get' in requestContext
  ? (requestContext as { get: (key: string) => unknown }).get('provider-mode')
  : undefined;
const isClaudeCode = providerMode === 'claude-code';

if (isClaudeCode && hasStructuredOutput) {
  // Delegate to generateWithRetry -- it uses agent.generate() which produces correct result.object.
  // The onTextChunk callback won't receive incremental chunks (generate is non-streaming),
  // but the full text is emitted once at completion for trace event consistency.
  const result = await generateWithRetry(agent, {
    prompt,
    options,
    timeoutMs,
    maxRetries,
    abortSignal: callerSignal,
    responseCheck: explicitResponseCheck,
  });

  // Emit the complete text as a single chunk so trace events still show reasoning
  if (onTextChunk && result.text) {
    onTextChunk(result.text);
  }

  // Convert Agent.generate() result to FullOutput shape for caller compatibility.
  // The generate result already has the same fields (text, object, steps, usage, etc.)
  return result as unknown as FullOutput;
}
```

This block goes after the options destructuring but before `let lastError`. The rest of the function (the streaming retry loop) remains completely unchanged -- it only executes for OpenRouter calls or non-structured Claude Code calls.

**Why this approach:**
- Zero changes to any step file (all 6 structured output call sites use streamWithRetry already)
- OpenRouter behavior is completely untouched (the fallback only triggers for claude-code)
- The FullOutput type is compatible with generate result since both have the same fields (.text, .object, .steps, .usage)
- onTextChunk still fires (with complete text rather than incremental chunks) so trace events work

**Type import:** Add `import type { FullOutput } from '@mastra/core/stream';` if not already present (it IS already imported at line 2).

**Important:** Do NOT change `generateWithRetry` at all. Do NOT change any step files. The fix is entirely within the `streamWithRetry` function body.
